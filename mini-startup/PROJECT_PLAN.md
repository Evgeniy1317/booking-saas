## 🏗️ АРХИТЕКТУРА

### Frontend (React + Vite + Tailwind)
```
src/
├── app/              # Основные страницы
│   ├── auth/        # Регистрация/Вход
│   ├── dashboard/   # Админка бизнеса
│   └── public/      # Публичная страница записи
├── components/       # Переиспользуемые компоненты
├── lib/             # Утилиты, конфиги
│   ├── supabase.ts  # Supabase клиент
│   └── utils.ts
├── hooks/           # Custom React hooks
└── types/           # TypeScript типы
```

### Backend (Supabase)
```
- Authentication (Email/Password)
- Database (PostgreSQL)
  - businesses (id, name, slug, owner_id, logo, colors, description, settings)
  - users (id, business_id, role, telegram_chat_id)
  - services (id, business_id, name, duration, price)
  - staff (id, business_id, name, services[]) - мастера
  - appointments (id, business_id, customer_id, service_id, staff_id, datetime, status)
  - customers (id, business_id, name, phone, email)
  - settings (business_id, notifications, telegram_enabled)
- Storage (логотипы, изображения)
- Edge Functions:
  - send-email (моментальные email уведомления)
  - telegram-webhook (обработка Telegram бота)
  - send-telegram (отправка в Telegram)
- Row Level Security (RLS) для multi-tenant
```

### Telegram Bot (отдельный сервис)
```
- Один бот для всех салонов
- Webhook от Telegram
- Определение салона по telegram_chat_id
- Команды:
  - /start - подключение салона
  - Уведомления владельцу
  - Отмена записи клиентом
```

### Deployment
- Frontend: Vercel
- Backend: Supabase Cloud
- Storage: Supabase Storage
- Telegram Bot: Vercel Edge Function или отдельный сервер

## 🎯 КЛЮЧЕВЫЕ РЕКОМЕНДАЦИИ

### 1. **Multi-tenant архитектура с первого дня**
- Использовать Row Level Security (RLS) в Supabase
- Каждый бизнес изолирован на уровне БД
- URL структура: `/b/[business-slug]` для публичных страниц

### 2. **Начать с минимального, но правильного**
- Не перегружать первый этап
- Но заложить правильную структуру БД
- Подумать о масштабировании заранее

### 3. **Приоритет: Работающий процесс записи**
- ЭТАП 2 - это сердце продукта
- **Клиент видит персонализированную страницу салона, а не платформу**
- Процесс: Услуга → Мастер → Дата/Время → Контакты → Подтверждение
- Максимально простой и понятный (3-4 клика)

### 4. **Безопасность данных**
- RLS политики для всех таблиц
- Валидация на клиенте И сервере
- Защита от SQL injection (Supabase делает это автоматически)

### 5. **UX для двух аудиторий**
- **Бизнес:** Простота управления, минимум кликов
- **Клиент:** Быстрая запись, понятный интерфейс

### 6. **Технологический стек - оптимален**
- React + Vite = быстрая разработка
- Supabase = backend из коробки (auth, DB, storage)
- Tailwind = быстрый UI
- Vercel = простой деплой

### 7. **Молдова как старт**
- Локализация (румынский/русский)
- Локальные платежные системы (если нужно)
- Понимание местного рынка

### 8. **Уведомления - критически важны**
- **Моментальные** - отправляются сразу после события
- Email обязателен (подтверждение, напоминание)
- Telegram для владельца - очень желательно
- Один бот для всех - масштабируемо
- Клиент может отменить через бота или по ссылке

### 9. **Публичная страница - это "сайт салона"**
- Клиент НЕ должен видеть, что это платформа
- Персонализация: название, логотип, цвета, текст
- Выглядит как официальная страница салона
- Это ключ к доверию и распространению
